# RULES.md — Règles de développement

## Code
- Un fichier = une responsabilité
- Commenter en français
- Tests avant merge sur main

## Git
- Branches : feature/nom-feature, fix/nom-bug
- Commits : feat: / fix: / docs: / refactor:
- Ne jamais committer de clés API

## IA
- Toujours inclure l'état complet dans chaque prompt
- Documenter les prompts qui fonctionnent dans prompts/
- Versionner les prompts comme du code
